# 真正有效的速度优化方案（从10MB/s提升到40MB/s+）

## 问题诊断

速度还是10MB/s，说明之前的优化没有真正生效。可能的原因：

1. **keyBy导致数据倾斜** - 所有数据都是同一个channel，keyBy后所有数据都到一个并行任务
2. **并行度没有真正生效** - LocalEnvironment可能限制了并行度
3. **批处理模式没有生效** - RuntimeExecutionMode.BATCH可能不兼容
4. **数据源还是串行的** - TdmsSampleSource可能没有真正并行

---

## 方案1：移除keyBy，使用rebalance强制并行（最重要⭐⭐⭐⭐⭐）

**问题**：`keyBy(s -> s.getChannel())` 如果所有数据都是同一个channel，会导致所有数据都到一个并行任务，完全串行！

**解决方案**：移除keyBy，改用rebalance强制分片

**代码改动**：
```java
// 当前（错误）：
DataStream<FilterResult> resultStream = stream
    .keyBy(s -> s.getChannel())  // 如果channel相同，所有数据到一个任务！
    .process(...)

// 改为（正确）：
DataStream<FilterResult> resultStream = stream
    .rebalance()  // 强制数据分片，真正并行
    .process(new GenericFilterProcessFunction(...))
    .setParallelism(parallelism);
```

**预期提升**：3-4倍（如果4核CPU）

**注意**：需要修改GenericFilterProcessFunction，不使用KeyedProcessFunction，改用ProcessFunction

---

## 方案2：强制提高并行度到CPU核心数（⭐⭐⭐⭐）

**问题**：`parallelism = CPU核心数 - 1` 可能还是太保守

**解决方案**：直接设置为CPU核心数，或者更高

**代码改动**：
```java
// 当前：
int parallelism = Math.max(1, Runtime.getRuntime().availableProcessors() - 1);

// 改为：
int parallelism = Runtime.getRuntime().availableProcessors(); // 使用所有核心
// 或者直接设置：
int parallelism = 8; // 如果知道是8核，直接设置
```

**预期提升**：1.5-2倍

---

## 方案3：移除批处理模式，使用流处理模式（⭐⭐⭐）

**问题**：`RuntimeExecutionMode.BATCH` 在LocalEnvironment可能不兼容，反而变慢

**解决方案**：移除批处理模式，使用流处理模式，但优化配置

**代码改动**：
```java
// 移除这行：
// env.setRuntimeMode(RuntimeExecutionMode.BATCH);

// 改为流处理优化：
env.setBufferTimeout(10); // 小缓冲区超时
env.getCheckpointConfig().disableCheckpointing();
```

**预期提升**：10-20%

---

## 方案4：简化GenericFilterProcessFunction，移除状态管理（⭐⭐⭐⭐）

**问题**：KeyedProcessFunction的状态管理有开销，批处理场景不需要

**解决方案**：改用ProcessFunction，不使用状态

**代码改动**：
- 将 `GenericFilterProcessFunction extends KeyedProcessFunction` 
- 改为 `extends ProcessFunction`
- 移除所有状态相关代码
- 使用本地变量维护窗口

**预期提升**：20-30%

---

## 方案5：减少窗口大小（⭐⭐⭐）

**问题**：窗口大小500ms可能太大，清理开销大

**解决方案**：减小窗口大小到100-200ms

**代码改动**：
```java
// 当前：
new GenericFilterProcessFunction(500, 3.0, true, algorithm)

// 改为：
new GenericFilterProcessFunction(100, 3.0, true, algorithm) // 窗口100ms
```

**预期提升**：10-15%

## 方案7：使用fromCollection而不是自定义Source（⭐⭐）

**问题**：自定义Source可能有同步开销

**解决方案**：直接使用fromCollection，Flink会自动优化

**代码改动**：
```java
// 当前：
DataStream<TdmsSample> stream = env.addSource(
    new TdmsSampleSource(data.getSamples()), 
    "tdms-source"
).setParallelism(parallelism);

// 改为：
DataStream<TdmsSample> stream = env.fromCollection(data.getSamples())
    .setParallelism(parallelism);
```

**预期提升**：5-10%
