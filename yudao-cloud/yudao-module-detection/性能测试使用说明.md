# TDMS文件处理速度性能测试使用说明

## 测试工具说明

我为您创建了两个性能测试工具：

### 1. QuickPerformanceTest.java - 快速单文件测试（推荐）

**功能**：快速测试单个TDMS文件的处理速度

**使用方法**：

```bash
# 方式1：使用默认参数（自动生成500万样本的测试文件）
cd yudao-module-detection-server
mvn exec:java -Dexec.mainClass="cn.iocoder.yudao.module.detection.QuickPerformanceTest" -Dexec.args=""

# 方式2：测试指定文件
mvn exec:java -Dexec.mainClass="cn.iocoder.yudao.module.detection.QuickPerformanceTest" -Dexec.args="path/to/your/file.tdms"

# 方式3：测试指定文件并使用指定算法
mvn exec:java -Dexec.mainClass="cn.iocoder.yudao.module.detection.QuickPerformanceTest" -Dexec.args="path/to/your/file.tdms KALMAN"
```

**支持的算法**：
- KALMAN
- LMS
- NLMS
- RLS
- MEAN
- MEDIAN
- GAUSSIAN
- FIR
- IIR
- 等等（所有FilterAlgorithm枚举值）

## 测试指标说明

### 性能指标

1. **解析吞吐量**：TDMS文件解析速度（MB/s）
   - 反映Python进程和I/O性能
   - 目标：>30 MB/s

2. **处理吞吐量**：Flink处理速度（MB/s）
   - 反映滤波算法和Flink性能
   - 目标：>40 MB/s（优化后）

3. **总吞吐量**：端到端处理速度（MB/s）
   - 包含解析+处理的总时间
   - 目标：>25 MB/s

### 时间占比

- **解析占比**：解析时间占总时间的百分比
- **处理占比**：Flink处理时间占总时间的百分比

**优化目标**：
- 如果解析占比>50%，说明解析是瓶颈，考虑优化Python进程
- 如果处理占比>70%，说明Flink处理是瓶颈，考虑提高并行度

## 常见问题

### Q1: 测试文件生成失败？
**A**: 确保已安装Python和nptdms库：
```bash
pip install nptdms numpy
```

### Q2: 并行度显示为1？
**A**: 检查CPU核心数，如果只有2核，并行度会自动设为1（保留1个核心给系统）

### Q3: 处理速度没有提升？
**A**: 
1. 检查CPU核心数（需要4核以上）
2. 检查内存是否充足
3. 查看日志中的并行度设置

### Q4: 如何测试自己的TDMS文件？
**A**: 
```bash
mvn exec:java -Dexec.mainClass="cn.iocoder.yudao.module.detection.QuickPerformanceTest" -Dexec.args="your_file.tdms KALMAN"
```

## 性能优化建议

根据测试结果：

1. **如果解析是瓶颈**（解析占比>50%）：
   - 考虑使用Python进程池
   - 优化Python脚本
   - 使用更快的I/O方式

2. **如果处理是瓶颈**（处理占比>70%）：
   - 提高并行度（需要更多CPU核心）
   - 优化滤波算法
   - 减少窗口大小

3. **如果都很快但总吞吐不高**：
   - 检查是否有其他开销（序列化、网络等）
   - 使用异步处理
