## 🎯 权限校验流程概览

### 完整调用链路

```
HTTP请求
    ↓
TokenAuthenticationFilter (Token认证过滤器)
    ↓
@PreAuthorize("@ss.hasPermission('system:user:create')") (Spring Security注解)
    ↓
SecurityFrameworkService (权限框架服务 - Bean名称: "ss")
    ↓
PermissionService (系统权限服务)
    ↓
查询用户角色 → 查询角色菜单 → 判断权限
    ↓
返回 true/false
```

## 🏗️ 核心组件

### 1. Controller层 - @PreAuthorize注解

```java
@RestController
@RequestMapping("/system/user")
public class UserController {
    
    @PostMapping("/create")
    @Operation(summary = "新增用户")
    @PreAuthorize("@ss.hasPermission('system:user:create')")  // 权限校验
    public CommonResult<Long> createUser(@Valid @RequestBody UserSaveReqVO reqVO) {
        Long id = userService.createUser(reqVO);
        return success(id);
    }
    
    @DeleteMapping("/delete")
    @PreAuthorize("@ss.hasPermission('system:user:delete')")  // 权限校验
    public CommonResult<Boolean> deleteUser(@RequestParam("id") Long id) {
        userService.deleteUser(id);
        return success(true);
    }
    
    @GetMapping("/page")
    @PreAuthorize("@ss.hasPermission('system:user:query')")   // 权限校验
    public CommonResult<PageResult<UserRespVO>> getUserPage(@Valid UserPageReqVO pageReqVO) {
        PageResult<AdminUserDO> pageResult = userService.getUserPage(pageReqVO);
        return success(BeanUtils.toBean(pageResult, UserRespVO.class));
    }
}
```

**注解说明**：
- `@ss` - Spring容器中名为"ss"的Bean（SecurityFrameworkService）
- `hasPermission('system:user:create')` - 调用该Bean的hasPermission方法
- 权限标识格式：`${系统}:${模块}:${操作}`
  - `system` - 系统标识
  - `user` - 用户模块
  - `create` - 创建操作

### 2. SecurityFrameworkService - 权限框架服务

#### 接口定义

```java
/**
 * Security 框架 Service 接口，定义权限相关的校验操作
 */
public interface SecurityFrameworkService {
    
    /**
     * 判断是否有权限
     */
    boolean hasPermission(String permission);
    
    /**
     * 判断是否有权限，任一一个即可
     */
    boolean hasAnyPermissions(String... permissions);
    
    /**
     * 判断是否有角色
     * 注意：角色使用的是 RoleDO 的 code 标识
     */
    boolean hasRole(String role);
    
    /**
     * 判断是否有角色，任一一个即可
     */
    boolean hasAnyRoles(String... roles);
    
    /**
     * 判断是否有授权范围
     */
    boolean hasScope(String scope);
    
    /**
     * 判断是否有授权范围，任一一个即可
     */
    boolean hasAnyScopes(String... scope);
}
```

#### 实现类

```java
@AllArgsConstructor
public class SecurityFrameworkServiceImpl implements SecurityFrameworkService {
    
    private final PermissionCommonApi permissionApi;  // RPC调用权限API
    
    /**
     * 权限缓存 - 1分钟过期
     * Key: <userId, List<permissions>>
     * Value: Boolean (是否有权限)
     */
    private final LoadingCache<KeyValue<Long, List<String>>, Boolean> hasAnyPermissionsCache = 
        buildCache(Duration.ofMinutes(1L), new CacheLoader<>() {
            @Override
            public Boolean load(KeyValue<Long, List<String>> key) {
                return permissionApi.hasAnyPermissions(
                    key.getKey(),  // userId
                    key.getValue().toArray(new String[0])  // permissions
                ).getCheckedData();
            }
        });
    
    /**
     * 角色缓存 - 1分钟过期
     */
    private final LoadingCache<KeyValue<Long, List<String>>, Boolean> hasAnyRolesCache = 
        buildCache(Duration.ofMinutes(1L), new CacheLoader<>() {
            @Override
            public Boolean load(KeyValue<Long, List<String>> key) {
                return permissionApi.hasAnyRoles(
                    key.getKey(), 
                    key.getValue().toArray(new String[0])
                ).getCheckedData();
            }
        });
    
    @Override
    public boolean hasPermission(String permission) {
        return hasAnyPermissions(permission);
    }
    
    @Override
    @SneakyThrows
    public boolean hasAnyPermissions(String... permissions) {
        // 特殊情况：跨租户访问
        if (skipPermissionCheck()) {
            return true;
        }
        
        // 获取当前登录用户ID
        Long userId = getLoginUserId();
        if (userId == null) {
            return false;
        }
        
        // 从缓存中获取权限校验结果
        return hasAnyPermissionsCache.get(new KeyValue<>(userId, Arrays.asList(permissions)));
    }
    
    @Override
    @SneakyThrows
    public boolean hasAnyRoles(String... roles) {
        // 特殊情况：跨租户访问
        if (skipPermissionCheck()) {
            return true;
        }
        
        // 权限校验
        Long userId = getLoginUserId();
        if (userId == null) {
            return false;
        }
        
        // 从缓存中获取角色校验结果
        return hasAnyRolesCache.get(new KeyValue<>(userId, Arrays.asList(roles)));
    }
}
```

**设计亮点**：
- ✅ **本地缓存**：使用Guava LoadingCache缓存权限判断结果
- ✅ **过期策略**：1分钟自动过期，平衡性能与实时性
- ✅ **跨租户支持**：特殊场景下跳过权限检查
- ✅ **RPC调用**：通过API调用权限服务

### 3. PermissionService - 系统权限服务

#### 核心权限判断方法

```java
@Service
@Slf4j
public class PermissionServiceImpl implements PermissionService {
    
    @Resource
    private RoleMenuMapper roleMenuMapper;
    @Resource
    private UserRoleMapper userRoleMapper;
    @Resource
    private RoleService roleService;
    @Resource
    private MenuService menuService;
    
    /**
     * 判断用户是否拥有任一权限
     * 
     * @param userId 用户ID
     * @param permissions 权限标识数组
     * @return 是否有权限
     */
    @Override
    public boolean hasAnyPermissions(Long userId, String... permissions) {
        // 1. 如果为空，说明已经有权限
        if (ArrayUtil.isEmpty(permissions)) {
            return true;
        }
        
        // 2. 获得当前登录的角色（启用状态）
        List<RoleDO> roles = getEnableUserRoleListByUserIdFromCache(userId);
        if (CollUtil.isEmpty(roles)) {
            return false;  // 没有角色，无权限
        }
        
        // 3. 遍历判断每个权限，如果有一个满足，说明有权限
        for (String permission : permissions) {
            if (hasAnyPermission(roles, permission)) {
                return true;
            }
        }
        
        // 4. 如果是超级管理员，也说明有权限
        return roleService.hasAnySuperAdmin(convertSet(roles, RoleDO::getId));
    }
    
    /**
     * 判断指定角色，是否拥有该 permission 权限
     * 
     * @param roles 指定角色数组
     * @param permission 权限标识
     * @return 是否拥有
     */
    private boolean hasAnyPermission(List<RoleDO> roles, String permission) {
        // 1. 根据权限标识，获取菜单ID列表（从缓存）
        List<Long> menuIds = menuService.getMenuIdListByPermissionFromCache(permission);
        
        // 2. 严格模式：如果权限找不到对应的Menu，认为没有权限
        if (CollUtil.isEmpty(menuIds)) {
            return false;
        }
        
        // 3. 判断角色是否拥有这些菜单
        Set<Long> roleIds = convertSet(roles, RoleDO::getId);
        for (Long menuId : menuIds) {
            // 获得拥有该菜单的角色编号集合（从缓存）
            Set<Long> menuRoleIds = getSelf().getMenuRoleIdListByMenuIdFromCache(menuId);
            
            // 如果有交集，说明有权限
            if (CollUtil.containsAny(menuRoleIds, roleIds)) {
                return true;
            }
        }
        
        return false;
    }
    
    /**
     * 获得用户拥有的角色，并且这些角色是开启状态的
     */
    @VisibleForTesting
    List<RoleDO> getEnableUserRoleListByUserIdFromCache(Long userId) {
        // 1. 获得用户拥有的角色编号（从缓存）
        Set<Long> roleIds = getSelf().getUserRoleIdListByUserIdFromCache(userId);
        
        // 2. 获得角色数组（从缓存）
        List<RoleDO> roles = roleService.getRoleListFromCache(roleIds);
        
        // 3. 移除被禁用的角色
        roles.removeIf(role -> !CommonStatusEnum.ENABLE.getStatus().equals(role.getStatus()));
        
        return roles;
    }
}
```

**权限判断流程**：

```
1. 检查用户是否有角色
   ↓ 有角色
2. 遍历每个权限标识 (如: system:user:create)
   ↓
3. 根据权限标识查询关联的菜单ID列表
   ↓
4. 遍历菜单，查询拥有该菜单的角色ID集合
   ↓
5. 判断用户的角色ID集合与菜单的角色ID集合是否有交集
   ↓ 有交集
6. 返回 true (有权限)
```

## 🔐 权限校验关键步骤详解

### 步骤1：用户 → 角色映射

```sql
-- system_user_role 表
SELECT role_id FROM system_user_role WHERE user_id = 1;
-- 结果: [1, 2] (用户拥有角色1和角色2)
```

### 步骤2：权限标识 → 菜单映射

## 📊 数据权限实现

除了功能权限，系统还支持**数据权限**（Data Permission），控制用户可以查看哪些数据。

### 数据权限类型（DataScopeEnum）

```java
public enum DataScopeEnum {
    ALL(1),              // 全部数据权限
    DEPT_CUSTOM(2),      // 指定部门数据权限
    DEPT_ONLY(3),        // 仅本部门数据权限
    DEPT_AND_CHILD(4),   // 本部门及子部门数据权限
    SELF(5);             // 仅本人数据权限
}
```

### 数据权限判断逻辑

```java
@Override
@DataPermission(enable = false)  // 关闭数据权限，避免递归
public DeptDataPermissionRespDTO getDeptDataPermission(Long userId) {
    // 1. 获得用户的角色
    List<RoleDO> roles = getEnableUserRoleListByUserIdFromCache(userId);
    
    DeptDataPermissionRespDTO result = new DeptDataPermissionRespDTO();
    
    // 2. 如果角色为空，则只能查看自己
    if (CollUtil.isEmpty(roles)) {
        result.setSelf(true);
        return result;
    }
    
    // 3. 遍历每个角色，计算数据权限
    for (RoleDO role : roles) {
        Integer dataScope = role.getDataScope();
        
        // 情况一：全部数据
        if (Objects.equals(dataScope, DataScopeEnum.ALL.getScope())) {
            result.setAll(true);
            continue;
        }
        
        // 情况二：指定部门数据
        if (Objects.equals(dataScope, DataScopeEnum.DEPT_CUSTOM.getScope())) {
            CollUtil.addAll(result.getDeptIds(), role.getDataScopeDeptIds());
            continue;
        }
        
        // 情况三：仅本部门数据
        if (Objects.equals(dataScope, DataScopeEnum.DEPT_ONLY.getScope())) {
            CollectionUtils.addIfNotNull(result.getDeptIds(), userDeptId.get());
            continue;
        }
        
        // 情况四：本部门及子部门数据
        if (Objects.equals(dataScope, DataScopeEnum.DEPT_AND_CHILD.getScope())) {
            CollUtil.addAll(result.getDeptIds(), 
                deptService.getChildDeptIdListFromCache(userDeptId.get()));
            CollUtil.addAll(result.getDeptIds(), userDeptId.get());
            continue;
        }
        
        // 情况五：仅本人数据
        if (Objects.equals(dataScope, DataScopeEnum.SELF.getScope())) {
            result.setSelf(true);
            continue;
        }
    }
    
    return result;
}
```

**返回结果示例**：

```java
DeptDataPermissionRespDTO {
    all: false,              // 不是全部数据
    self: false,             // 不仅限本人
    deptIds: [10, 20, 30]    // 可查看部门10、20、30的数据
}
```

## 💾 缓存机制

### 三级缓存架构

```
┌─────────────────────────────────────────┐
│  Level 1: Guava本地缓存 (1分钟)          │
│  - hasAnyPermissionsCache               │
│  - hasAnyRolesCache                     │
└────────────────┬────────────────────────┘
                 ↓ 未命中
┌─────────────────────────────────────────┐
│  Level 2: Redis缓存 (长期)               │
│  - USER_ROLE_ID_LIST:{userId}           │
│  - MENU_ROLE_ID_LIST:{menuId}           │
│  - PERMISSION_MENU_ID_LIST:{permission} │
└────────────────┬────────────────────────┘
                 ↓ 未命中
┌─────────────────────────────────────────┐
│  Level 3: MySQL数据库                    │
│  - system_user_role                     │
│  - system_role_menu                     │
│  - system_menu                          │
└─────────────────────────────────────────┘
```

### 缓存失效策略

#### 1. 角色菜单变更时

```java
    // 缓存自动失效
```

## 🚀 性能优化要点

### 1. 多级缓存

- **Guava本地缓存**：1分钟过期，避免频繁RPC调用
- **Redis缓存**：长期缓存，避免数据库查询
- **懒加载**：只在第一次访问时加载

### 2. 批量查询

```java
// ❌ 不好的做法：循环查询
for (Long roleId : roleIds) {
    List<RoleMenuDO> list = roleMenuMapper.selectListByRoleId(roleId);
}

// ✅ 好的做法：批量查询
List<RoleMenuDO> list = roleMenuMapper.selectListByRoleId(roleIds);
```

### 3. 缓存预热

```java
// 系统启动时预加载常用权限
@PostConstruct
public void init() {
    List<MenuDO> menus = menuService.getMenuList();
    // 预加载到缓存
}
```

### 4. 避免缓存穿透

```java
// 即使查询结果为空，也缓存起来
@Cacheable(value = "cache", key = "#id")
public List<RoleDO> getRoles(Long id) {
    List<RoleDO> roles = roleMapper.selectListByUserId(id);
    return roles == null ? Collections.emptyList() : roles;
}
```

## 🔒 安全注意事项

### 1. 敏感操作强制权限校验

```java

    userService.deleteUser(id);
    return success(true);

// ✅ 安全：添加权限校验

    userService.deleteUser(id);
    return success(true);
```

## 📝 总结

### 权限校验的核心特点

1. **三层架构**：
   - Controller层：`@PreAuthorize` 注解声明权限
   - Service层：`SecurityFrameworkService` 权限判断
   - Data层：`PermissionService` 具体实现

2. **RBAC模型**：
   - User → Role → Menu
   - 用户拥有角色，角色拥有菜单，菜单对应权限

3. **多级缓存**：
   - Guava本地缓存（1分钟）
   - Redis缓存（长期）
   - MySQL数据库

4. **数据权限**：
   - 支持5种数据范围
   - 自动SQL过滤

5. **高性能**：
   - 缓存命中率高
   - 批量查询
   - 懒加载
