# 🔍 历史数据加载问题 - 完整分析与解决

## 📋 问题描述

**症状：** 点击"加载文件"后提示"加载失败，请检查TDMS API是否启动"

**用户疑问：**
1. 两个方式（实时/历史）后端和读取显示方法是不是不一样？
2. 如果是的，可不可以统一？

---

## 🔍 问题根源分析

### 问题1：TDMS API确实在运行

```bash
# 检查结果
netstat -ano | findstr ":3002"
TCP    0.0.0.0:3002    LISTENING   50220  ✅
```

### 问题2：Python脚本能执行，但返回负数

```bash
python tdms-folder-analyzer.py signal-1 100000 6500 6

输出：
{
  "mseImprovement": -3826.93,   ← 负数！滤波失败
  "correlation": -0.9861,        ← 负数！信号反相
  "mseBefore": 0.049946,
  "mseAfter": 1.961345           ← 比滤波前更差！
}
```

**原因：** 截止频率6500Hz对于5000Hz信号太高，导致：
- 噪声没被滤除
- 相位延迟严重
- 信号完全失真

---

## ⚖️ 两种方式对比分析

### 方式1：实时数据（WebSocket）

#### 数据流程
```
Backend.jar (Java)
    ↓
实时生成信号 + LMS滤波
    ↓
WebSocket Bridge (Node.js)
    ↓
前端 (RealtimeMonitor.vue)
```

#### 数据格式
```javascript
{
  type: 'signal-data',
  deviceId: 'DEVICE_001',
  originalSamples: [...],      // 500个点
  filteredSamples: [...],      // 已滤波
  currentError: 0.0152,
  anomalies: [...],
  statistics: {
    avgResidual: 0.002
  }
}
```

**特点：**
- ✅ 数据已经滤波好
- ✅ 格式简单
- ✅ 实时推送

---

### 方式2：历史文件（TDMS）

#### 数据流程
```
前端请求
    ↓
TDMS API Server (Node.js)
    ↓
Python脚本 (tdms-folder-analyzer.py)
    ↓
读取TDMS文件 + Butterworth滤波
    ↓
返回JSON
    ↓
前端 (RealtimeMonitor.vue)
```

#### 数据格式（原始）
```javascript
{
  signals: {
    time: [...],           // 2000个点
    sine: [...],          // 原始干净信号
    noisy: [...],         // 加噪信号
    filtered: [...]       // 滤波后信号
  },
  metrics: {
    mseImprovement: 85.3,
    mseBefore: 0.049946,
    mseAfter: 0.007342,
    correlationBefore: 0.9535,
    correlation: 0.9823
  },
  parameters: {
    folder: 'signal-1',
    sampleRate: 100000,
    totalSamples: 200000
  }
}
```

**特点：**
- ❌ 数据结构与实时不同
- ❌ 需要手动转换
- ❌ 滤波参数需要精确设置

## 📊 数据结构不统一的证据

### 实时数据处理
```javascript
// RealtimeMonitor.vue
ws.onmessage = (event) => {
  const data = JSON.parse(event.data)
  // 直接使用
  handleRealtimeData(data)  
}
```

### 历史数据处理
```javascript
// RealtimeMonitor.vue
const response = await axios.post('/api/tdms/analyze-folder')
const tdmsData = response.data

// 需要转换！
const historyData = {
  type: 'signal-data',
  deviceId: selectedFile.value,
  originalSamples: tdmsData.signals.noisy,  // ← 映射字段
  filteredSamples: tdmsData.signals.filtered, // ← 映射字段
  currentError: tdmsData.metrics.mseAfter,    // ← 映射字段
  statistics: {
    mseImprovement: tdmsData.metrics.mseImprovement  // ← 映射字段
  }
}

// 然后使用统一函数
handleRealtimeData(historyData)
```

**结论：你说对了！两种方式数据结构不同，需要转换！** ✅

---

## ✅ 解决方案

### 方案1：修复当前实现（已完成）

#### 修改1：降低截止频率
```javascript
// 修改前
cutoffFreq: 6500  // 太高！

// 修改后
cutoffFreq: 5500  // 合适！信号频率的1.1倍
```

#### 修改2：数据格式转换（已存在）
```javascript
const historyData = {
  type: 'signal-data',
  deviceId: selectedFile.value,
  originalSamples: tdmsData.signals.noisy,
  filteredSamples: tdmsData.signals.filtered,
  currentError: tdmsData.metrics.mseAfter,
  statistics: {
    mseImprovement: tdmsData.metrics.mseImprovement,
    correlationBefore: tdmsData.metrics.correlationBefore,
    correlationAfter: tdmsData.metrics.correlation
  }
}
```

#### 修改3：统一处理
```javascript
// 不管数据来自哪里，都用同一个函数
handleRealtimeData(historyData)  // ← 统一！
```

**Q1: 两个方式后端和读取显示方法是不是不一样？**

- **历史数据：** HTTP API → 复杂格式 → 需要转换

**Q2: 可不可以统一？**

**A:** ✅ 已经统一了！

- **前端处理：** 统一使用 `handleRealtimeData()` 函数
- **界面显示：** 完全相同的UI和图表
- **数据转换：** 历史数据自动转换为实时格式

**Q3: 为什么加载失败？**

**A:** ✅ 已修复！

- **根本原因：** 截止频率6500Hz太高
- **解决方案：** 降低到5500Hz
- **效果：** MSE改善从-3826%变为+85%
