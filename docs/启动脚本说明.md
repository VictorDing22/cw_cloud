# 🚀 **启动脚本使用说明**

## ✨ **所有脚本已优化：无交互、无弹窗**

所有启动脚本已经过优化，实现：
- ✅ **无需按键确认** - 自动运行或超时等待
- ✅ **无弹窗运行** - 后台启动，输出重定向到日志文件
- ✅ **自动化启动** - 一键启动所有服务

---

## 🎯 **推荐启动方式**

### **方式1: 完整滤波系统（推荐）**

```bash
start-filter-system.bat
```

**包含组件：**
- Kafka消息队列
- backend.jar滤波服务
- FloatData数据生成器
- WebSocket Bridge
- 前端Vue监控界面

**启动时间：** 约2-3分钟
**自动打开：** http://localhost:3000/realtime/filter

---

### **方式2: backend.jar滤波服务**

```bash
start-backend-filter.bat
```

**功能：**
- 启动backend.jar（端口8080）
- 配置Kafka输入/输出主题
- 访问：http://localhost:8080/cal

**日志文件：** backend.log

---

### **方式3: 卡尔曼滤波异常检测**

```bash
启动异常检测系统.bat
```

**功能：**
- 卡尔曼滤波算法
- 异常检测引擎
- WebSocket实时推送
- 前端监控界面

**日志文件：**
- kalman-service.log
- websocket-bridge.log
- vue-dev.log

---

### **方式4: 完整CW_Cloud系统**

```bash
start-all.bat
```

**包含所有微服务：**
- MySQL、Redis、Nacos、Kafka
- Gateway、System、Infra等微服务
- backend.jar
- FloatData
- 前端Vue

**启动时间：** 约5-6分钟

---

## 📁 **日志文件位置**

所有服务的日志都输出到项目根目录：

| 服务 | 日志文件 | 说明 |
|------|---------|------|
| backend.jar | `backend.log` | 滤波服务日志 |
| WebSocket Bridge | `websocket-bridge.log` | 数据桥接日志 |
| 卡尔曼滤波 | `kalman-service.log` | 滤波算法日志 |
| FloatData | `netty-server.log` | 数据生成器日志 |
| 前端Vue | `vue-dev.log` | 前端开发服务器日志 |

## 🔍 **故障排查**

### **查看日志**

所有服务都有对应的日志文件，遇到问题时：

```bash
# Windows
type backend.log
type websocket-bridge.log

# 或使用文本编辑器打开日志文件
```

### **检查端口占用**

```bash
netstat -ano | findstr ":8080"
netstat -ano | findstr ":8081"
netstat -ano | findstr ":9092"
netstat -ano | findstr ":3000"
```

### **停止所有服务**

```bash
# 停止Node.js服务
taskkill /F /IM node.exe

# 停止Java服务
taskkill /F /IM java.exe
```

## 🎯 **快速测试流程**

### **测试backend.jar滤波**

1. 启动系统：
   ```bash
   start-filter-system.bat
   ```

2. 访问backend界面：
   ```
   http://localhost:8080/cal
   ```

3. 点击"新开WebSocket"，选择设备，生成数据

4. 观察波形图显示滤波效果

5. 访问前端监控：
   ```
   http://localhost:3000/realtime/filter
   ```

6. 点击"开始监控"查看实时数据

---

### **测试卡尔曼滤波异常检测**

1. 启动系统：
   ```bash
   启动异常检测系统.bat
   ```

2. 自动打开浏览器访问：
   ```
   http://localhost:3000/realtime/filter
   ```

3. 登录（admin/admin123）

4. 点击"开始监控"

5. 查看：
   - 实时波形对比
   - 卡尔曼滤波残差
   - 异常检测结果表格
   - 异常通知弹窗

## 💡 **最佳实践**

1. **首次启动**
   - 使用 `start-filter-system.bat` 启动完整系统
   - 等待所有服务完全启动（约2-3分钟）
   - 查看日志文件确认无错误

2. **日常开发**
   - 根据需要启动特定服务
   - 使用日志文件调试问题
   - 定期清理旧日志文件

3. **问题排查**
   - 先检查端口占用
   - 查看对应的日志文件
   - 确认依赖服务（Kafka、MySQL等）正常运行
